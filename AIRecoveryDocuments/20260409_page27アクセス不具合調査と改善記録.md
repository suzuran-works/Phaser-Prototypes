# Page27 アクセス不具合調査と改善記録（2026-04-09）

## 発生した不具合
- トップ画面に Page27 への遷移リンクは表示されるが、ビルド成果物に `page27/index.html` が出力されず、本番配信時に Page27 が 404 になる。

## 原因
- `vite.config.ts` の `build.rollupOptions.input` に `page27` エントリが未登録だった。
- そのため、`vite build` 実行時に Page27 がマルチページビルド対象から外れていた。

## 改善内容
- `vite.config.ts` の入力設定へ `page27: resolve(__dirname, 'page27', 'index.html')` を追加した。
- `npm run build` を実行し、`dist/page27/index.html` が生成されることを確認した。

## 再発防止策
- 新しい `pageXX/` を追加した際は、同じコミットで `vite.config.ts` の `input` 登録有無をチェックリスト化して確認する。
- リリース前に `dist/pageXX/index.html` の生成有無を対象ページ分だけ確認する。
