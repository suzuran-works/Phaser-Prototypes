## 実装概要

### ゲームコンセプト

アイソメトリック（クォータービュー）視点の温泉リゾートを見守る放置・タップ系ゲーム。
うさぎたちが自律的に訪れ、温泉を楽しんで去っていく。プレイヤーは落とされたコインをタップして集める。

### 技術的アプローチ

**ビュー方式**: アイソメトリックグリッド（11×11タイル）
- グリッド→スクリーン変換: `sx = ISO_OX + (gx - gy) * TILE_HW`, `sy = ISO_OY + (gx + gy) * TILE_HH`
- 深度ソート: オブジェクトのスクリーンY値を `setDepth()` に使用することで手前/奥の重なりを自動制御
- 温泉プール: `SPRING_DEPTH = SPRING_SY + 1` を境界値として、Y<境界のキャラは温泉の奥に見える

**主要定数**:
- タイル: TILE_HW=45, TILE_HH=22.5
- グリッド中心: (5,5) → スクリーン(540, 505)
- 温泉プール描画: 石枠 + 水面楕円 + 湯気Tween

**ウサギのAI**:
- 状態: `walking_in` → `soaking` → `leaving` → `done`
- 入浴スロット6箇所（奥2・手前4）を排他管理
- スポーン: グリッド端10箇所からランダムに選択
- 入浴中: 揺れアニメ（sin波）・吹き出し（7〜15秒ごと）・飲酒（10〜22秒ごと）
- 退場: コイン1〜3枚をバウンスアニメで落下

**ペア会話**:
- 入浴中の2匹を15〜28秒ごとに選出
- 両者に吹き出しを表示（3.5秒後自動消去）

**コイン収集**:
- 退場時にコインを落下（バウンスEase）
- 6.2秒の寿命（最後1.2秒でフェードアウト）
- タップ範囲44px以内で回収、+1スコア表示

## 要点メモ（後続改修用）

- 温泉の深度境界 `SPRING_DEPTH = 506` を変更する場合は `SPRING_SY` 連動で調整すること
- スロット座標はアイソメトリック座標系で定義されており、石枠視覚との整合を確認すること
- ペア会話は `pairTimer` で制御。同時発火を防ぐため `!r.bubble` チェック済み
- 絵文字フォントサイズはモバイルとデスクトップで見え方が異なる場合がある
