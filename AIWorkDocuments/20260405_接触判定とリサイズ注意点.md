# Phaserにおける接触判定とリサイズ時の注意点

## 背景
画面リサイズ（レスポンシブ対応）そのものが接触判定ロジックを壊すわけではないが、
表示スケールと判定領域の更新が不一致だと「当たり判定がずれた」状態が発生する。

## 代表的なずれの原因
- 見た目だけを `setDisplaySize` / `setScale` で変更し、Physics Body サイズを更新していない。
- `Scale.FIT` やカメラズーム後に、入力座標の解釈（画面座標とワールド座標）が混在している。
- リサイズでUIの見た目のみ再配置し、インタラクティブ領域（hitArea）を再設定していない。
- スプライトの原点（origin）変更後に、Body の offset を再調整していない。

## 実装時のチェックポイント
1. **座標系を統一する**
   - 接触判定はワールド座標ベースで扱い、画面座標依存の比較を避ける。
2. **見た目変更後に判定領域を再計算する**
   - Arcade Physics 利用時は `refreshBody()`、必要に応じて `body.setSize()` / `body.setOffset()` を実行する。
3. **リサイズイベントで再設定を集約する**
   - `this.scale.on('resize', ...)` で、再配置・再スケール・判定更新をまとめて行う。
4. **入力判定を再設定する**
   - `setInteractive()` の対象が表示サイズ変更の影響を受ける場合、hitAreaを明示して再設定する。
5. **デバッグ可視化で確認する**
   - Physics debug や hitArea の可視化により、リサイズ前後でのズレ有無を確認する。

## 推奨運用フロー
- 基準解像度を定義する。
- 画面サイズに応じて表示スケールを更新する。
- 同タイミングで Physics Body / hitArea を更新する。
- 最後に、実機（縦長・横長・高DPR）で入力と衝突判定を確認する。

## 最小チェックリスト（PRレビュー用）
- [ ] リサイズ時に `resize` ハンドラで再配置と再スケールを実施している。
- [ ] 見た目変更後に Body サイズ・offset が更新されている。
- [ ] Input hitArea が表示サイズと一致している。
- [ ] カメラズーム時のポインター座標変換を確認している。
- [ ] スマホ縦横でタップ判定ずれがないことを確認している。
